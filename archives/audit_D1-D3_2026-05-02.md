# Cross-cutting observations

1. **D1 is roughly accurate.** Minor right-edge SH 5 cleanup only. The photo-verified SH 2 stacking annotation is a great pattern — apply this technique to other shelves.

2. **D2 is OVER-CODED on top 2 shelves.** The actual store has SH 5 ~50% empty (rack with promo signage) and SH 4 ~40% empty (more signage). The deck currently renders a fully-stocked D2, which contradicts what the user can see in his store. This is the inverse of the user's stated complaint — we have phantom product where there's actually empty rack + signage. The "Super Good · Enjoy Them Here" and "Welcome to Happy Place" promotional cards should be encoded as either (a) empty/dashed-capacity tiles or (b) explicit `{name:'Promo card', status:'remove'}` placeholders.

3. **D3 has a NEW-in-BEFORE bug.** Multiple SKUs marked `status:'new'` are coded into the BEFORE shelfData but logically shouldn't render in the BEFORE pane — they're aspirational adds that exist only in AFTER. Either:
   - Filter NEW status out at render time when drawing the BEFORE pane, OR
   - Move NEW tiles to a separate `afterShelves` data structure.

4. **Missing Monster Original on D3 SH 5.** The photo clearly shows 3 facings of Monster Original (green M, black can) anchoring the left side of the top shelf. The current code has Celsius dominating SH 5 with no Monster Original — Monster is currently coded only on SH 4. Need to add it to SH 5 BEFORE (and decide whether to keep there or consolidate to SH 4 in AFTER).

5. **Photo-verification annotations.** D1 SH 2 has a great inline comment noting "PHOTO-VERIFIED: 12oz cans, 12 across, STACKED 2-HIGH, 7 deep." Applying this same kind of annotation to every shelf as we audit would make the BEFORE/AFTER reasoning more transparent and auditable.
